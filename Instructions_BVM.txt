    made one change to streamline the process.<br>
    <br>
    I followed the setup instructions. Things omitted or glossed over:<br=

    directory.)<br>

    * What about tex-squares.sty? (Following Guy's advice from message

    that requires it.)<br>
    <br>
    * Following Guy's instructions from earlier today, I edited <font
      face=3D"monospace">%LOCALAPPDATA%\Programs\MiKTeX\miktex\tex\latex\=
standalone\standalone.cfg</font>,
    inserting<br>
    <blockquote><font face=3D"monospace">\standaloneconfig{convert=3D{con=
vertexe=3D{magick
        convert}}}</font><br>
    </blockquote>
    =C2=A0 (I put this near the beginning. By the way, as of ImageMagick
    version 7, it's no longer necessary to say "convert", so this could
    just be ...<font face=3D"monospace">{convertexe=3D{magick}}</font>.)<=
br>
    <br>
    Once I did all this, I achieved success, and got relatively few
    error messages in the log:<br>
    <blockquote><font face=3D"monospace">Transcript written on
        sgltndl.log.<br>
        pdflatex: major issue: So far, you have not checked for MiKTeX
        updates.<br>
        convert: profile 'icc': 'RGB ': RGB color space not permitted on
        grayscale PNG `sgltndl.png' @
        warning/png.c/MagickPNGWarningHandler/1748.<br>
        <br>
        Class standalone:<br>
        Output written on sgltndl.png.<br>
        =C2=A0) )<br>
        (\end occurred when \ifx on line 2 was incomplete)<br>
        No pages of output.</font><br>
    </blockquote>
    As for the next step, the suggestion to crop the png using the
    Windows Photos app, why not just use ImageMagick, as long as we have
    it installed anyway and we're sitting at the command line? This
    would be simpler than opening Photos (tho maybe you'd want to make a
    little .bat file to simplify life):<br>
    <br>
    =C2=A0=C2=A0 =C2=A0<font face=3D"monospace">magick sgltndl.png -trim =
-transparent
      white -bordercolor white -border 10x10 sgltndl.png</font><br>
    <p>But wait. Since pdflatex is already producing output using
      ImageMagick, why not do the cropping in the pdflatex step to begin
      with and save the second step? I know zero about Tex, but I didn't
      see a way I could add convert arguments later in the pipe just by
      setting the convertexe variable. So instead I went into
      standalone.cls and made what seemed like the obvious edit:</p>
    <blockquote><font face=3D"monospace">\sa@convertoption{imagemagick}[]=
{%<br>
        =C2=A0=C2=A0=C2=A0 \def\sa@convert@command{\convertexe\space -den=
sity
        \density\space -units PixelsPerInch \infile\space
        \ifx\size\empty\else -resize \size\fi\space -quality 90 <b>-color=
space
          RGB </b><b>-trim -transparent white -bordercolor white
          -border 10x10</b> \outfile}%<br>
        =C2=A0=C2=A0=C2=A0 \let\sa@convert@pageoffset\m@ne<br>
        }</font><br>
    </blockquote>
    (There's now one more option in this command line, "-colorspace
    RGB", which is a kludgy way of getting rid of the only significant
    error in the log shown above, "RGB color space not permitted". As a
    side effect, it also makes the output png slightly smaller. The
    border-adding commands aren't strictly necessary, but make the
    picture in Word a little more pleasant when it has a selection
    border around it.)<br>
    <br>
    And this works great, no extra manual cropping step needed. Guy,
    maybe you could recommend a cleaner way of doing this.<br>
    <br>
    As for inserting the pictures in the Word doc, my work flow is
    slightly simpler.<br>
    <ul>
      <li>insert a new paragraph (blank line)</li>
      <li>click Insert &gt; Pictures, finding the file on disk</li>
      <li>leave the default "In Line with Text" set</li>
      <li>adjust the scale if desired ("Size and Position" in the
        context menu)</li>
      <li>make the inserted paragraph centered, and adjust its
        before/after leading as desired</li>
    </ul>
    95% seems to be about the right scale to match the existing diagrams
    in the C-3A doc.<br>
